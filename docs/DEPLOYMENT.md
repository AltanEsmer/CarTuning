## 1. Prerequisites

Before deploying, ensure the following are satisfied:

### Software

- **Python 3.8+** installed and on PATH
- **pytest** installed (`pip install pytest`)
- **Razer Synapse 4** installed and fully updated
- All project dependencies installed (`pip install -r requirements.txt`)

### Validation

- All unit tests pass:
  ```bash
  pytest tests/
  ```
- Pattern visualization passes the smell test for all weapons
- Statistical validation is within tolerance (see `docs/start.md` for baseline values)

### Hardware

- Razer mouse connected and recognized by Synapse
- Mouse configured to:
  - **DPI**: 800 (reference DPI for all weapon data)
  - **Polling Rate**: 1000Hz

## 2. Synapse Profile Setup

### 2.1 Create the RUST Profile

1. Open **Razer Synapse 4**
2. Navigate to **Dashboard** → select your mouse device
3. Click **Profiles** → **+** to create a new profile
4. Name it: **RUST**
5. Optionally link to the Rust game executable (not required for training)
6. Configure base settings:

| Setting | Value | Notes |
|---------|-------|-------|
| DPI | **800** | Reference DPI — all weapon data is calibrated to this |
| Polling Rate | **1000Hz** | Ensures timing precision for macro delays |
| Acceleration | **OFF** | Must be disabled for consistent compensation |
| Angle Snapping | **OFF** | **Critical** — interferes with micro-movements |

> ⚠️ **Angle Snapping**: If this is left on, Synapse will "smooth" the small mouse movements generated by macros, destroying the compensation pattern. Always verify it is disabled.

### 2.2 Button Mapping

Assign each mouse button to its corresponding weapon macro toggle:

| Button | Assignment | Purpose |
|--------|-----------|---------|
| LMB | Default (click) | Fire weapon — **NOT** macro-bound |
| RMB | Default (ADS) | Aim down sights |
| Button 4 (side) | Toggle: `RECOIL_AK` | Activate AK-47 compensation |
| Button 5 (side) | Toggle: `RECOIL_LR300` | Activate LR-300 compensation |
| Button 6 (side/scroll) | Toggle: `RECOIL_MP5` | Activate MP5 compensation |
| Scroll Click | Toggle: `RECOIL_M249` | Activate M249 compensation |
| DPI Button | Toggle: `RECOIL_CUSTOM_SMG` | Activate Custom SMG compensation |

> **Note**: Thompson is low-priority and can share a binding with scroll modifiers or be omitted entirely.

To assign a button:
1. Click the button in the Synapse device view
2. Select **Macro** from the function list
3. Choose the corresponding `RECOIL_*` macro
4. Set activation to **While Assigned Button is Held**

### 2.3 The Toggle System

**Critical design decision**: Macros activate on the **toggle button**, not on LMB.

#### How It Works

```
1. Press and HOLD Button 4 (e.g., AK toggle)
2. While holding Button 4, hold LMB to fire
3. Macro compensates recoil while BOTH buttons are held
4. Release Button 4 to deactivate compensation
5. LMB continues to work normally for non-automatic weapons
```

#### Why This Design

| Benefit | Explanation |
|---------|-------------|
| **Clean LMB** | No interference with pistols, shotguns, building, looting, etc. |
| **Instant weapon switching** | Change which side button you hold — no profile reloading |
| **No mid-game profile swaps** | All weapons are available simultaneously |
| **No accidental activation** | Compensation only runs when you deliberately hold the toggle |

## 3. Importing Generated Macros

### 3.1 Macro File Structure

The Python generator outputs **two files per weapon** to `src/output/`:

- **`.rzn`** — Synapse 4 native import format (XML). This is what you import directly.
- **`.txt`** — Human-readable reference. For reviewing and debugging, not for import.

The `.rzn` file follows the Synapse 4 XML macro schema:

```xml
<?xml version="1.0" encoding="utf-8"?>
<Macro xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
  <Name>RECOIL_AK47</Name>
  <Guid>unique-guid-per-generation</Guid>
  <MacroEvents>
    <MacroEvent>
      <Type>3</Type>
      <Delay>102</Delay>
      <MouseMovement>
        <MouseMovementEvent>
          <Type>3</Type>
          <X>-1</X>
          <Y>6</Y>
        </MouseMovementEvent>
      </MouseMovement>
    </MacroEvent>
    <!-- ...continues for all shots -->
  </MacroEvents>
  <IsFolder>false</IsFolder>
  <FolderGuid>00000000-0000-0000-0000-000000000000</FolderGuid>
</Macro>
```

Each `<MacroEvent>` contains:
- **`<Type>3</Type>`** — Mouse movement event type
- **`<Delay>`** — Humanized delay in milliseconds before this movement
- **`<X>`** / **`<Y>`** — Relative mouse movement in pixels (humanized, DPI-scaled)

### 3.2 Importing into Synapse 4

Direct `.rzn` import — no manual entry needed:

1. Open **Synapse 4** → Select your mouse device
2. Go to the **Macro** tab (left sidebar)
3. Click the **three-dot menu** (⋮) in the macro list
4. Select **Import**
5. Navigate to `src/output/` and select `RECOIL_AK47.rzn`
6. The macro appears in your macro list, ready to assign
7. Assign it to the corresponding toggle button (see Section 2.2)
8. Set the macro playback to: **While Assigned Button is Held**

Repeat for each weapon:
- `RECOIL_AK47.rzn`
- `RECOIL_LR300.rzn`
- `RECOIL_MP5.rzn`
- `RECOIL_M249.rzn`
- `RECOIL_CUSTOM_SMG.rzn`
- `RECOIL_THOMPSON.rzn`

### 3.3 Regenerating Macros

Regenerate macros periodically for fresh humanization. Each run produces unique sequences (new GUIDs, new randomized values):

```bash
# Single weapon
python src/generator.py --weapon ak47

# All weapons at once
python src/generator.py --weapon all
```

After regenerating, re-import the updated `.rzn` files into Synapse 4 (Section 3.2). The old macros can be deleted from Synapse's macro list before importing fresh ones.

## 4. DPI Considerations

All weapon data is calibrated for **800 DPI**. If you use a different DPI, pull values must be scaled.

### Scaling Table

| Your DPI | Scale Factor | Action |
|----------|-------------|--------|
| 400 | 2.0× | Double all pull values |
| 800 | 1.0× | No changes needed (reference DPI) |
| 1200 | 0.67× | Reduce all pull values by 33% |
| 1600 | 0.5× | Halve all pull values |

### Automatic Scaling

The generator handles DPI scaling automatically with the `--dpi` flag:

```bash
python src/generator.py --weapon ak47 --dpi 1600 --output src/output/
```

The formula applied internally:

```
scaled_value = base_value × (800 / your_dpi)
```

> ⚠️ **Do not** manually scale values if you are using the `--dpi` flag — the generator does this for you.

## 5. First-Run Testing Checklist

### 5.1 Pre-Game Validation

Complete all of these before entering any server:

- [ ] Run unit tests: `pytest tests/`
- [ ] Generate fresh macros for all weapons
- [ ] Visualize at least 3 sprays per weapon (check for obvious errors)
- [ ] Confirm Synapse **RUST** profile is active
- [ ] Confirm DPI is set to **800** (or correct value with `--dpi` flag used)
- [ ] Confirm polling rate is **1000Hz**
- [ ] Test each toggle button activates the correct macro (check Synapse macro tab — it highlights active macros)

### 5.2 In-Game Testing (Build/Training Server)

Join a **build server** or **aim training server**. For each weapon, run through:

#### Test 1: Static Spray

Stand still. Spray a full magazine at a wall at medium range.

- Does the crosshair stay roughly on target?
- Does the pattern look natural (scattered, not a laser line)?
- Does accuracy degrade toward the end of the magazine? (expected — later shots are harder to compensate)

#### Test 2: Toggle Switching

Rapidly switch between weapons and toggle buttons.

- Do the correct macros activate for each weapon?
- Is there any delay or interference between toggles?
- Does releasing the toggle button cleanly deactivate compensation?
- Does LMB work normally with no toggle held?

#### Test 3: ADS (Aim Down Sights)

Test compensation while aiming down sights.

- ADS typically reduces recoil in Rust — compensation may be too strong
- You may need separate ADS profiles in the future
- **Note**: The initial version does **not** account for ADS multiplier

#### Test 4: Movement

Spray while strafing (A/D keys).

- Macro should still compensate (it only generates mouse movement)
- Verify no conflicts between WASD input and macro activation
- Check that movement doesn't cause toggle buttons to accidentally release

### 5.3 Success Criteria

For each weapon, confirm:

- [ ] Crosshair stays within **~10% of target** at 50m range
- [ ] Spray pattern shows **visible scatter** (not a perfect grouping — that looks unnatural)
- [ ] **No identical sprays** when testing 5+ times (humanization is working)
- [ ] Toggle system works **without interference** across all weapons
- [ ] **No noticeable input lag** or stuttering during macro execution

## 6. Troubleshooting

| Problem | Likely Cause | Fix |
|---------|-------------|-----|
| Macro doesn't activate | Toggle button not assigned | Check Synapse button mapping (Section 2.2) |
| Compensation too strong | DPI mismatch (your DPI > 800) | Verify DPI setting; regenerate with `--dpi` flag |
| Compensation too weak | DPI mismatch (your DPI < 800) or wrong profile | Verify DPI; check correct Synapse profile is active |
| Crosshair drifts left/right | X-axis calibration off | Adjust X-drift values in weapon JSON (`src/weapons/`) |
| Macro feels "robotic" | Humanization variance too low | Increase variance ranges in the weapon JSON config |
| Macro feels "sloppy" | Humanization variance too high | Decrease variance ranges in the weapon JSON config |
| Input lag during macro | Polling rate too low | Set to 1000Hz in Synapse device settings |
| Macro fires extra shots | Wrong activation mode | Set to **"While Held"**, not **"Toggle On/Off"** |
| Macro active on wrong weapon | Button assignment error | Re-check button → macro mapping in Synapse |
| Synapse doesn't detect mouse | Driver/firmware issue | Reinstall Synapse 4; update mouse firmware |

## 7. Maintenance

### Periodic Regeneration

Regenerate macros periodically to prevent pattern staleness and ensure humanization variance stays fresh:

```bash
python src/generator.py --weapon all --output src/output/
```

**Recommended frequency**: Before each training session, or at minimum weekly.

### Profile Backup

Export your Synapse profile regularly:

1. Open **Synapse** → **Profiles**
2. Right-click the **RUST** profile → **Export**
3. Save the exported file to the `backups/` folder in the project directory

```
CarTuning/
├── backups/
│   ├── RUST_profile_2026-03-30.synapse4
│   └── RUST_profile_2026-04-15.synapse4
└── ...
```

### Weapon Balance Updates

When a Rust game update changes weapon recoil patterns:

1. **Update** the relevant weapon JSON in `src/weapons/`
2. **Regenerate** macros:
   ```bash
   python src/generator.py --weapon all --output src/output/
   ```
3. **Re-run validation**:
   ```bash
   pytest tests/
   ```
4. **Re-import** updated macros into Synapse (Section 3.2)
5. **Re-test** on a training server (Section 5.2)

### Version Tracking

Keep notes on which Rust game version each weapon JSON was calibrated against. This makes it easy to identify which weapons need recalibration after a game patch.
